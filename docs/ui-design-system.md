# NexoRig UI 规范：Operational Clarity

本规范用于前台日报、分析统计、月度报表、HSSE、系统后台和登录页。目标是在不改变业务计算、数据接口和 Excel 模板的前提下，让所有页面使用同一种企业级视觉语言。

## 1. 设计原则

1. **作业优先**：页面先突出当前任务、异常和下一步操作，再展示辅助信息。
2. **高密度但可扫描**：保留专业系统的数据密度，以层级、留白、冻结列和状态标签提升阅读效率。
3. **一套组件，多种业务**：日报、分析、月报和后台只能组合公共组件，不再为单个页面重新定义按钮、输入框、卡片或导航。
4. **状态明确**：选中、禁用、只读、加载、成功、警告和错误必须同时用颜色与文字表达。
5. **正式报表独立**：月报操作区遵循系统规范，报表画布严格保留 Excel 模板字体、边框、行高和导出格式。

## 2. 页面脉络

| 页面族 | 典型页面 | 统一骨架 |
| --- | --- | --- |
| 日报工作台 | 钻井、完井、修井 | 井列表 + 日历 + 当前井摘要 + 上传记录 |
| 日报详情 | 三类日报详情 | 页面标题 + 状态指标 + 分区表单 + 明细表 |
| 分析统计 | 时效、生产、NPT | 筛选栏 + 指标条 + 图表/主表 + 追溯详情 |
| 正式月报 | 附表4、5、6、工作量 | 系统筛选工具栏 + 独立报表画布 + Excel 导出 |
| HSSE | 填报、驾驶舱、报表 | 风险总览 + 异常清单 + 闭环任务；未开发功能使用标准空状态 |
| 系统后台 | 主数据、AI、权限、日志 | 后台标题 + 概览指标 + 主表 + 抽屉/弹窗编辑 |

## 3. 设计变量

### 颜色

- 主导航：`#082B4B`
- 页面标题：`#172B3D`
- 主要操作：`#1768B0`
- 安全/成功：`#0F7F76`
- 警告：`#A86408`
- 错误/NPT：`#B83232`
- 页面背景：`#F1F5F9`
- 模块背景：`#FFFFFF`
- 分隔线：`#DCE5EE`

颜色不得作为唯一状态提示；状态必须同时包含文字或图标。

### 字体

- 中文界面：`Noto Sans SC / Source Han Sans SC / PingFang SC / Microsoft YaHei`
- 数据：等宽数字字体回退栈
- 正文：14px；辅助文字：12px；极少量表格辅助文字：11px
- 页面标题：20px；模块标题：14px；关键指标：20–24px
- Excel 正式报表继续使用模板指定宋体，不受系统界面字体覆盖

### 间距、圆角和控件

- 间距只使用：4、8、12、16、24、32px
- 圆角只使用：6、8、12px
- 紧凑控件高度：32px；标准控件高度：40px
- 常规模块不使用阴影；阴影仅用于弹窗、下拉浮层和提示

## 4. 公共组件规范

### 导航

- 侧栏展开宽度 216px，窄屏 204px。
- 菜单分组使用安静的文字层级；当前页面使用完整蓝色底，不允许同一时刻出现多个激活项。
- 前台与后台使用同一图标、间距和选中规则。

### 页面标题栏

- 左侧：面包屑和页面标题。
- 右侧：页面操作、全局语言和账户信息。
- 页面操作与全局操作在视觉上保持间距，禁止把所有按钮挤成一组。

### 按钮

- 主按钮：页面唯一的主要提交或导入动作。
- 次按钮：查询、重置、返回、普通导出。
- 文字按钮：表格行内低风险动作。
- 危险按钮：删除、清空、停止任务，必须二次确认。

### 表单

- 标签始终位于控件上方；单位写入标签或字段说明，不放在用户输入值中。
- 只读字段使用浅灰底；禁用字段保留可读对比度。
- 错误在字段附近给出具体原因，保存失败时同时显示页面级提示。

### 表格

- 业务操作表使用浅色表头、轻分隔线和行悬停。
- 长表支持横向滚动；关键列优先冻结。
- 数字右对齐并使用等宽数字；操作列保持固定宽度。
- 正式月报画布不套用业务表格样式。

## 5. 多语言和可访问性

- 语言选择在全系统保持；西语布局至少预留 40% 文本扩展空间。
- 所有可操作元素必须有清晰的键盘焦点。
- 状态不只依赖颜色；图标需要可访问名称。
- 低于 900px 时导航变为顶部块；低于 760px 时表单和摘要卡片改为单列或双列。
- 动画遵循系统“减少动态效果”设置。

## 6. 代码治理

- `styles.css` 保留业务组件和专用报表布局。
- `ui-system.css` 是唯一公共视觉层，禁止在业务页面继续新增按钮、输入框、导航、卡片和通用表格的重复定义。
- 新页面优先复用现有类；只有业务结构确实不同才新增页面专属类。
- 不再加载旧的 `liquid-glass.js` 装饰脚本，减少无业务价值的运行时 DOM 注入和样式覆盖。
